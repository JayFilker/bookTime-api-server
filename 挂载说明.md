


# Render 挂载说明

## 首次部署前（本地操作）

如果 `src/data/` 下已有 JSON 文件可跳过此步。否则先在本地爬取一次数据：

```bash
npm run crawl
```

爬取完成后会在 `src/data/` 生成四个文件：

- `books.json` — 书籍列表（21 本，覆盖 7 个分类）
- `chapters.json` — 章节列表（全部章节条目）
- `first-chapters.json` — 每本书第一章的正文内容
- `home.json` — 首页推荐/最新入库数据

将这四个文件连同代码一起提交到 Git 仓库，Render 部署时会直接打包进去。

```bash
git add src/data/books.json src/data/chapters.json src/data/first-chapters.json src/data/home.json
git commit -m "feat: 更新预存爬取数据"
git push
```

## Render 部署配置

项目根目录已有 `render.yaml`，Render 会自动读取，无需手动配置构建命令。

| 配置项 | 值 |
|---|---|
| 构建命令 | `npm install && npm run build` |
| 启动命令 | `npm start` |
| Node 版本 | 自动检测 |

### 环境变量

在 Render 控制台的 **Environment** 页面手动填写以下变量：

| 变量名 | 说明 | 示例 |
|---|---|---|
| `NODE_ENV` | 固定填 `production` | `production` |
| `JWT_SECRET` | 自定义密钥，随机字符串即可 | `a8f3k2m9x...` |
| `JWT_EXPIRES_IN` | Token 有效期 | `7d` |
| `BASE_URL` | Render 分配的域名，部署后在控制台查看 | `https://novel-api-server.onrender.com` |

> `BASE_URL` 必须填，否则上传的头像/图片 URL 拼接会出错。格式为 `https://` 开头，不带尾部斜杠。

## 生产环境行为说明

部署到 Render 后，服务**不会**发起任何爬虫请求，所有数据来自预存 JSON：

| 接口 | 数据来源 |
|---|---|
| 书籍列表 / 详情 | `books.json` |
| 章节列表 | `chapters.json` |
| 第一章正文 | `first-chapters.json` |
| 其他章节正文 | 返回乱码占位内容 |
| 首页推荐 | `home.json` |
| 搜索 | 从 `books.json` 按标题/作者模糊匹配 |

---

## 本地开发

```bash
npm run dev
```

本地运行时 `NODE_ENV=development`，所有接口实时爬取，不使用预存数据，功能与部署前完全相同。

---

## 更新预存数据

当需要刷新书籍内容时，在本地重新爬取并推送：

```bash
npm run crawl
git add src/data/*.json
git commit -m "feat: 更新预存数据"
git push
```

Render 检测到推送后会自动重新构建部署。
